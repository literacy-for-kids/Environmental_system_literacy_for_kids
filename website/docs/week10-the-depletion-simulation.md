# Week 10: The Resource Pool Game
*Unit 3: System Limits and Load Balancing*

This week is a tabletop game. The game is the lesson.

The game is the **Resource Depletion Simulation**. Two or more players share a renewable resource pool — fish, trees, groundwater, whatever you want to call it. Each round, players decide how much to harvest. Whatever is left regenerates at a fixed rate. The question is: can the players harvest sustainably, or will the pool collapse?

The game illustrates two of the most important ideas in resource economics:

- **Maximum sustainable yield (MSY)** — the largest amount you can harvest each round without depleting the pool over time. Equal to the regeneration rate.
- **Tragedy of the commons** — a structural pressure that pushes individual players to over-harvest, even when they know it will collapse the pool.

Played well, this game produces real emotional moments. Students *feel* the temptation to over-harvest. They negotiate, sometimes successfully, sometimes not. They see collapse happen with their own tokens.

---

:::info Facilitator Snapshot

## For Younger Learners (Ages 8–9)

:::info Adapting This Week
**Simplest version of the concept:** "We're going to play a game where we share a pile of fish. If we take too many, we run out. If we take just the right amount, we can keep playing forever."

**What to shorten or skip:**
- Start the pool smaller (10 tokens) and use slower regeneration (1 token per round) for clearer dynamics.
- Skip silent rounds. Younger learners get more out of negotiating openly.

**Adapting the activities:**
- Use real-life framing: "We're sharing a pond full of fish. How many fish can each of us catch this year?"

**Journal alternatives:** Draw the game's pool over time — how many fish at the start of each round, with the player's harvests.

**What success looks like:** The learner can say "if we take too many, we run out, but if we leave enough for them to make new ones, we can keep eating fish forever."
:::

:::info For Ages 10–12
- Calculate MSY explicitly each round. With a 25% regeneration rate on a pool of 20, MSY = 5 per round (or 2.5 per player in a 2-player game).
- Run multiple rounds with different rules: silent, then with negotiation, then with binding agreements, then with private side-deals allowed.
- Compare outcomes. Discuss why some structures support cooperation and others don't.
:::

## The Game: Resource Depletion Simulation (Full Rules)

### Setup (2-player version)

- Pool starts at **20 tokens**.
- Regeneration rate: **25% per round, rounded down, minimum 0**.
- Each player can secretly choose to harvest **0 to 5 tokens** per turn.
- Game lasts **10 rounds**.

### Each Round

1. **Announce the pool size.** Both players see how many tokens are in the pool.
2. **Secret harvest decisions.** Each player writes down (on a slip of paper or behind a hand) the number of tokens they want to harvest. **No talking** during this step in the basic version.
3. **Reveal.** Both players reveal their numbers simultaneously.
4. **Apply the harvest.** Remove the total harvested tokens from the pool.
5. **Check for collapse.** If the pool hits 0 at this point, the fishery has collapsed. Game over. **Nobody wins.** Whatever each player has harvested up to this point is lost — the moral being that even past harvests depended on a functioning ecosystem.
6. **Regeneration.** Add 25% of the remaining pool, rounded down (minimum 0). So if 12 tokens remain, add 3 (12 × 0.25 = 3). If 4 remain, add 1 (4 × 0.25 = 1). If 1 remains, add 0 (1 × 0.25 = 0.25, rounded down).
7. **Record.** Each player keeps their harvested tokens in a private pile.
8. **Next round** starts with the new pool size.

### Win Condition

After 10 rounds, the player with the **most tokens** wins — **unless** the pool collapsed at any point, in which case nobody wins.

### Variants

**4–6 player version.** Same rules, but:
- Pool starts at **40 tokens** instead of 20.
- Each player can still harvest 0–5 per turn.

**Negotiation allowed.** Players are allowed to talk before each round but cannot bind each other to any agreement. They are still each making private harvest decisions.

**Binding agreements.** Players can sign written agreements before each round. If someone breaks one, they have to forfeit half their accumulated tokens. (This represents the "regulation with enforcement" condition.)

**Private deals.** Players can quietly pair off, make side agreements, etc. Closer to real-world political dynamics.

### The MSY Calculation

For the 2-player game with a 20-token pool and 25% regeneration:

- Regeneration on a full pool of 20: 20 × 0.25 = 5 tokens per round.
- That's the **Maximum Sustainable Yield (MSY)**: 5 tokens per round shared between the players.
- In a 2-player game, that's an average of **2.5 tokens per player per round**.
- Over 10 rounds, that's **25 tokens per player** — if they cooperate perfectly.

If players overharvest, the pool shrinks. The next round's regeneration is smaller (25% of a smaller number). They've now reduced future income. If they continue to over-harvest from a smaller pool, the system spirals down to collapse.

## Guided Session 1
### Play the Game
#### Learning Goal
By the end of this session, the student can:

- play the Resource Depletion Simulation
- calculate the maximum sustainable yield for the game's setup
- describe how individual harvest decisions interact to determine system outcome

---

#### Activities

**1. Play the First Game (Silent)**

Set up the 2-player version (or 4-6 if you have more players). Use the basic rules. **No talking** between players. Each player makes private decisions.

Play through 10 rounds. Record the data:

| Round | Pool Before | Player 1 | Player 2 | Pool After Harvest | + Regen | Pool After Regen |
|-------|-------------|----------|----------|---------------------|---------|------------------|
| 1 | 20 | | | | | |
| 2 | | | | | | |
| ... | | | | | | |

---

**2. Debrief the First Game**

After 10 rounds (or earlier collapse), review the data together.

- Did the pool collapse?
- How many tokens did each player end with?
- What was the player's *strategy*? (Conservative? Aggressive? Mixed?)
- Did either player feel tempted to take more than the sustainable amount? Why?
- How did each player feel about the other player?

The most common outcomes:
- One or both players harvest aggressively in early rounds (often 4-5 tokens), the pool drops fast, regeneration shrinks, and the pool collapses by round 5–7. Nobody wins.
- Both players play conservatively but slightly above MSY, the pool slowly shrinks, and ends up smaller than it started. Players end with maybe 15–25 tokens each.
- Rarely on first play: both players land on the MSY by intuition, the pool stays at 20, and each player ends with ~25 tokens. (This is the optimal outcome.)

---

**3. Calculate the MSY**

After play, calculate it together:

> "Each round, the pool grows by 25% of whatever was left. If the pool stays at 20, that's 5 tokens of growth. With two players, that's 2.5 tokens each per round, or 25 total per player over 10 rounds — if you cooperated perfectly."

Now compare to actual results.

> "How did your actual harvest compare to MSY? If you overshot, what was the consequence?"

## Guided Session 2
### The Tragedy of the Commons
#### Learning Goal
By the end of this session, the student can:

- explain why individually rational decisions can collectively destroy a shared resource
- describe the tragedy of the commons as a structural feature, not a moral failing
- identify mechanisms that allow cooperation despite the structural pressure

---

#### Activities

**1. Play Round 2 (Negotiation Allowed)**

Reset the pool to 20. Play again — but this time, **players are allowed to talk** before each round.

Common pattern: players negotiate a shared rule ("let's each take 2 or 3 tokens"). They mostly stick to it. The pool stays close to 20. Both players end with ~25 tokens. Both win the satisfying way.

Or sometimes one player defects mid-game and takes 5 to get ahead. Then trust breaks down, and the rest of the game becomes a race to deplete the pool before the other player can.

---

**2. The Tragedy of the Commons**

After the second round, introduce the concept:

> "The thing you just experienced has a name. It's called the **tragedy of the commons**, from a famous essay by Garrett Hardin in 1968 (though the idea is older)."

Walk through the logic:

- A pasture is open to all the local farmers. Each farmer can graze as many cows as they want.
- If a farmer adds one more cow, they get the full benefit of that cow's milk and meat.
- The cost (slight overgrazing of the shared pasture) is spread across all the farmers.
- So for any individual farmer, **adding another cow is rational** — their gain is larger than their share of the cost.
- But if every farmer makes the same rational decision, the pasture is overgrazed and ruined for everyone.

> "The tragedy is not that anyone is greedy or stupid. It's that the *structure of the situation* rewards individual over-use, even when collective over-use destroys the resource."

This is the same pattern as the depletion game with no negotiation.

---

**3. What Can Break the Trap?**

If the structure of the situation produces the tragedy, what changes the structure?

**Mechanism 1: Communication.** Players who can talk often find cooperative agreements. (Like Round 2 of the game.)

**Mechanism 2: Binding agreements (regulation).** Players who can make enforceable commitments don't have to trust each other — they have to follow the rule. Real-world examples: fishing quotas, hunting seasons, water rights.

**Mechanism 3: Private ownership.** If the pasture is split into private lots, each farmer manages their own. They can't overgraze a shared resource because there isn't one anymore. Works for some resources (land), poorly for others (atmosphere, ocean fish).

**Mechanism 4: Long-term repeated interaction.** Players who know they'll play many times with the same partners have incentive to build trust. Reputation matters. Defection in one round affects future rounds.

**Mechanism 5: Smaller groups with shared norms.** Small communities often manage commons effectively through social pressure and local knowledge (Elinor Ostrom won a Nobel Prize in economics for documenting this).

If you have time, play a third round with one of these mechanisms in place. Discuss how it changes the game.

---

**4. Real-World Examples**

Map the game to real-world commons:

| Real-world commons | "Pool" | "Regeneration rate" | "Players" |
|---|---|---|---|
| Atlantic cod fishery | Cod stock | ~10–20% per year (varies) | Commercial fishing fleets |
| Ogallala Aquifer | Groundwater | Very slow (~hundreds of years to refill) | Midwest farmers, towns |
| Atmosphere (CO₂ uptake) | Capacity to absorb extra CO₂ | Slow (decades-to-centuries) | Every emitter on Earth |
| Public Wifi at a coffee shop | Bandwidth | Constant flow | Every customer |
| A shared printer at school | Paper / toner / time | Refilled by staff | All students |

Most of these are real cases where overshoot has happened or is happening. The atmospheric one (CO₂) is the largest commons problem ever attempted.

## Independent Practice

### Goal

Identify and analyze a commons in your own environment.

### Activities

**1. Spot the Commons**

Find three commons-like resources in your life. They can be tiny (the snacks in a shared pantry) or large (your local park). For each one:

- Who are the "players"?
- What is the "pool"?
- What is the "regeneration rate" (if any)?
- Is the resource being managed sustainably, or is it being overused?

Examples:

- **A neighborhood park** — pool: open space, condition. Players: residents. Regeneration: slow (maintenance budget). Often well-managed.
- **A common pantry at home** — pool: snacks. Players: family. Regeneration: weekly shopping. Often a tragedy in small ways.
- **A shared streaming service password** — pool: simultaneous-stream limit. Players: friends and family. Regeneration: instant when someone logs off.
- **A school playground** — pool: equipment, time. Players: students. Regeneration: maintenance, but also the kids themselves taking care of it.

---

**2. Design a Rule**

Pick one of the commons from your list that is being overused. Design one rule (any mechanism — communication, regulation, ownership, etc.) that could fix it without abolishing the commons.

Examples:

- "Whoever takes the last cookie has to buy the next box."
- "Family meetings on Sunday: everyone declares their snack for the week."
- "Phone in a basket during dinner so nobody hoards the household's attention."

---

**3. Terrarium Check**

Week 7 of terrarium observation. Open the Systems Log:

- Day count: __
- Condensation, soil, plants — any changes?
- This week, notice: your terrarium has no players competing for the pool. The plants share the same soil, water, and CO₂, but they're not making decisions. The system stays stable because no one is over-harvesting.

### Systems Log

> **Today's System:** Resource Depletion Simulation
>
> **First-game results:** ____
>
> **MSY for the game:** ____
>
> **What changed when we could negotiate:** ____
>
> **A commons in my real life:** ____
>
> **Rule I would design to manage it:** ____
>
> **Terrarium update:** ____

**Sentence starters for younger learners:**
- "If we share, we can have ___ fish forever. If we don't, we run out after ___ rounds."
- "A commons in my life is ___."

### Reflection Questions

- In the silent game, what made you tempted to take more than your share?
- In the negotiation game, did you trust the other players? Why or why not?
- If the same two players played the game 10 times in a row, would you expect their behavior to change over the 10 games?
- What real-world commons could be saved by communication alone? Which need stronger mechanisms (regulation, ownership, etc.)?

## Check for Understanding

After this week, check whether the learner can:

1. **Calculate MSY:** "If a pond has 100 fish and they breed at 10% per year, what's the maximum sustainable harvest?" (Looking for: 10 fish per year.)
2. **Describe the tragedy:** "Why might individual rational decisions destroy a shared resource?" (Looking for: each individual benefits from over-using, but the cost is spread across everyone.)
3. **Name one mechanism:** "Name one thing that helps players manage a commons sustainably." (Looking for: communication, regulation, ownership, repeated games, small-group norms — any of these.)

If the learner can do at least 2 of these, they're ready for Week 11.

## Pause and Notice

:::note What Matters Here
Ask:

> "In the silent game, you might have over-harvested even when you knew the math. Why?"

The honest answer: the structure of the situation pulled you toward over-harvesting. You couldn't trust the other player. You wanted to make sure you got *your* share before they took it.

> "That isn't a moral failing. That's the structure speaking through you. Real fishing fleets, real farmers, real corporations experience the same pressure on a much larger scale. They aren't villains — they are responding to the same incentives."

The engineering question is: **how do we restructure the situation so that the structure rewards sustainable behavior instead of unsustainable behavior?**

**This week's takeaway:** The tragedy of the commons is a structural pattern, not a moral one. Different structures produce different outcomes — even with the exact same players.
:::
